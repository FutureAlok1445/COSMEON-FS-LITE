Satellites orbit at ~550km altitude with 90-minute orbital periods. Each 
ground-station communication window is only 5–10 minutes per pass. Traditional 
TCP/IP assumes continuous end-to-end connectivity — it fails completely. 
COSMEON implements Store-and-Forward (DTN) behavior: queue chunks, transmit 
when the link is available, resume from the interruption point. NASA has used DTN 
operationally since 2024 with 100% delivery success on the PACE mission (34M+ 
bundles delivered). 
 
P2 
Radiation 
Bit 
Corruption 
Cosmic rays cause Single Event Upsets (SEUs) that randomly flip individual bits in 
satellite memory. A file stored naively will silently corrupt over time — with no visible 
error, no warning. COSMEON detects and auto-corrects with multi-level SHA-256 
verification at every single read and write operation, catching what other systems 
miss entirely. The Bit Rot chaos scenario demonstrates this live.

CubeSats have severe Size, Weight, and Power (SWaP) constraints. Simple 3x

Amazon S3, Google Cloud Storage, HDFS, Backblaze, and Facebook f4 — 
achieves identical 2-node fault tolerance with only 50% storage overhead. ACM 
2024 confirms RS codes are the most widely deployed erasure codes in production.

Pillar 2 — Orbital Topology-Aware Distribution 
The 6 satellite nodes are organised into 3 orbital planes (2 satellites per plane). The hard architectural 
rule: a data chunk and its corresponding RS parity chunk are NEVER placed on the same orbital plane. 
This guarantees that a full-plane blackout — whether from a solar storm or communication failure 
across an entire orbital shell — cannot destroy more chunks than RS coding can recover from. No 
other hackathon team considers orbital topology as a distribution constraint.

Each node has an orbit timer — a 120-second countdown simulating the satellite's Line-of-Sight (LOS) 
window with the ground station. When a node's timer drops below 30 seconds, the system 
automatically and proactively migrates the most-requested chunks from that node to the node with the 
highest remaining timer. The node does NOT fail — you see the failure coming and act before it 
happens. This is proactive fault tolerance, not reactive recovery. 
 
Pillar 4 — DTN Store-and-Forward Queue 
Inspired by NASA's Bundle Protocol (BPv7), operationally deployed in 2024 on the PACE mission with 
100% delivery success over 34 million bundles. When a target node is temporarily unreachable, chunks 
are queued in a per-node transmission buffer (/dtn_queue/{node_id}.json). A background asyncio task 
polls all offline nodes every 5 seconds. On node recovery, the queue flushes automatically in priority 
order (parity chunks first), with each delivery verified. This models real orbital communication windows 
— the system does not fail when a satellite is temporarily silent; it waits and delivers. 
 
Pillar 5 — Chaos Engineering Mode 
Chaos Engineering is a real production discipline pioneered by Netflix (Chaos Monkey). COSMEON 
implements 4 one-button disaster scenarios — each mapping to a documented real-world space 
hazard — that play out with live animations, log updates, and metric changes across the Mission 
Control dashboard. Judges watch the system break and recover in real time. 
 
Pillar 6 — Shannon Entropy Auto-Rebalancing 
After any topology change, the system computes Shannon Entropy H = -sum(p_i * log2(p_i)) across all 
nodes, where p_i is the fraction of total chunks on node i. If entropy drops below 0.85, the rebalancer 
identifies hotspot nodes (above-average chunk count) and migrates chunks to underloaded nodes until 
entropy exceeds 0.90. The entropy gauge on the dashboard climbs back in real time — this is real 
information theory, not a cosmetic metric. 
 
Pillar 7 — Live Accuracy Metrics Dashboard 
Six real distributed systems metrics update continuously via WebSocket: MTTDL, Storage Efficiency 
Ratio, Chunk Distribution Entropy, Reconstruction Latency Breakdown, Integrity Verification Pass Rate, 
and Cache Hit Rate. Every formula is sourced from peer-reviewed distributed systems research. This 
transforms the demo from a pretty UI into a scientifically credible measurement instrument.
